# Smart Task Manager

A full-stack, AI-powered Smart Task Manager application that allows users to manage their daily tasks and receive curated daily briefings utilizing the Google Gemini AI.

## 🚀 Features

- **Secure Authentication**: JWT-based login and signup securing isolated task lists for each user.
- **Task Management**: Create, view, update, and delete tasks dynamically.
- **AI Daily Briefing**: Integrated with the `@google/genai` model to generate personalized, friendly daily briefings based on your pending tasks. Graceful error handling for API outages and network issues.
- **Stunning UI**: Responsive interface built with Next.js, customized with a premium dark-mode aesthetic, glassmorphism impacts, and beautiful layout structures.

## 💻 Tech Stack

- **Frontend**: Next.js 16, React 19, TailwindCSS 4, Framer Motion, Lucide React
- **Backend**: Node.js, Express.js 5, TypeScript
- **Database**: PostgreSQL with Prisma ORM 7
- **AI**: Google Gemini AI (`@google/genai`)

## 📋 Prerequisites

Before you begin, ensure you have the following installed:
- [Node.js](https://nodejs.org/en/) (v18+)
- [PostgreSQL](https://www.postgresql.org/) database setup and running locally or on a cloud platform
- [Google Gemini API Key](https://aistudio.google.com/app/apikey)

## 🛠️ Setup Instructions

### 1. Database & Backend Setup

1. Open a terminal and navigate to the backend directory:
   ```bash
   cd backend
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Create a `.env` file in the `backend` directory based on the environment requirements. You will need:
   ```env
   # .env
   PORT=4000
   DATABASE_URL="postgresql://<USER>:<PASSWORD>@<HOST>:<PORT>/<DATABASE>?schema=public"
   JWT_SECRET="your_highly_secure_jwt_secret"
   GEMINI_API_KEY="your_google_gemini_api_key"
   ```
4. Push Prisma schema to your PostgreSQL database:
   ```bash
   npx prisma db push
   # Alternatively, you can run `npx prisma migrate dev`
   ```
5. Generate the Prisma client:
   ```bash
   npm run build
   # Or run `npx prisma generate` directly
   ```
6. Start the development server:
   ```bash
   npm run dev
   ```
   *The backend should now be running on `http://localhost:4000`.*

### 2. Frontend Setup

1. Open a new terminal and navigate to the frontend directory:
   ```bash
   cd frontend
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Create a `.env.local` file in the `frontend` directory:
   ```env
   # .env.local
   NEXT_PUBLIC_API_URL=http://localhost:4000/api
   ```
4. Start the frontend development server:
   ```bash
   npm run dev
   ```
   *The frontend should now be running on `http://localhost:3000`.*

## 💡 Usage

Navigate to `http://localhost:3000` in your browser.
1. **Sign Up / Login** to your personal account.
2. In the Dashboard, add a few tasks focusing on your plans for the day.
3. Check the right sidebar where your **Focus for Today** will be automatically summarized and generated by the AI briefing manager based on your pending tasks.

## ⚙️ Error Handling
The app utilizes custom-built mapping to smoothly handle scenarios where the AI endpoint may face downtime or rate limits:
- Built-in rate limit management.
- Failsafes connecting to your backend fallback messages for heavy traffic instances (`503 Service Unavailable`).
- Database mapping errors and robust retry configurations handled visually gracefully.
